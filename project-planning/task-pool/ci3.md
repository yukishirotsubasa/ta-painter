# ci3 — proxy 健康檢查排程（已取消，由 data8 取代）

## 狀態

**已取消（2026-07-23）。** 本任務不再執行，已從 `task-list.md` 移除，改由 [data8](data8.md) 以不同方向解決同一則技術債（TPEx／Yahoo 反爬蟲規則變動導致 proxy 失效）。

## 取消理由

原方案是加 GitHub Actions cron 定期 `curl` 兩個 proxy 端點、失敗時告警。經評估後改變方向：上游資料源出問題時**會立即影響到使用者**，而週期性檢查最快也要等到下一個排程週期才發現，對使用者當下遇到的失敗毫無幫助；即使告警成功，使用者端仍只看得到一句原始錯誤訊息，不知道發生什麼事、也不知道該不該回報。

因此改為在**使用端**處理：資料層對錯誤分類，只有在判定為「上游被擋／掛掉」時，才在錯誤訊息下方追加提示，告知資料源可能已失效、請聯絡製作者。詳見 [data8](data8.md)。

## 原始內容（存檔）

新增 GitHub Actions 排程（cron），定期 `curl` TPEx / Yahoo 兩個 proxy 端點（TPEx 個股、Yahoo `.TW`/`.TWO`），驗證回應正常；失敗時發通知（如自動開 issue 或其他告警），及早發現上游反爬蟲規則變動導致 proxy 失效（而非等使用者回報）。
